# VELO 2.0 Local Deployment Preparation

This directory contains blueprints, templates, and planning documents for migrating VELO 2.0 from cloud/preview environments to a local Ubuntu server.

## Status: DORMANT / PREPARATION ONLY
These files do not affect the active running platform. They serve as a configuration blueprint for a future manual migration. No scripts in this directory run automatically. Buildy remains the active production environment.

## VELO 2.0 Autonomy Preparation Index
The following items are prepared as dormant artifacts for Ubuntu:
- **Backend Deployment**: Blueprints for Node/Deno services and process management.
- **Database Migration**: Schema mapping and data transformation specs.
- **Environment Templates**: Secure placeholder templates for local variables.
- **Connector Activation**: Logic for arming platform-specific profiles.
- **Credential Activation**: Safe handling specs for local vault injection.
- **Browser Automation**: Blueprints for headless execution (Puppeteer/Playwright).
- **Execution Engines**: Pseudocode for real-world action processing.
- **Autopilot Autonomy**: Logic for self-directed mission execution.
- **System Services**: Config templates for PM2, NGINX, Redis, PostgreSQL.
- **Logging & Monitoring**: Black Box and system health tracking specs.
- **Activation Switch**: Safety gate for full autonomy unlock.
- **Safety Core**: Pre-execution validation and boundary enforcement.
- **Evidence Packets**: Formal review records for dry-run validation proof.
- **Fallback Logic**: Redundancy and error handling protocols.
- **Validation Planner**: Dry-run drills for Data, Providers, and Browsers.
- **Optimization Roadmap**: Prioritized performance and UX enhancements.

## NEVER AUTO-RUN POLICY
To ensure platform safety and production stability:
- **No Ubuntu Connection**: No attempts to reach or connect to local hosts from Buildy.
- **No Local Deployment**: No automatic installation or setup of local services.
- **No Credential Activation**: No raw secrets are stored or transmitted in these templates.
- **No Connector Writes**: No platform profiles are modified or activated automatically.
- **No Browser Automation**: No local headless browsers are triggered.
- **No Active Trading/Finance**: No financial operations execute without manual host-level activation.

## Migration Principles
- **Free-First**: Preference for local AI models (Ollama, LM Studio) and open-source tools.
- **Autonomy**: Offline-capable operation with local process management.
- **Safety**: Isolated preparation, non-destructive exports, and fully reversible steps.
- **Ubuntu 24.04 LTS**: Targeted host environment.

## Execution Requirements
Ubuntu remains dormant until an administrator manually enters the activation phrase: **"Enable Full Autonomy (Ubuntu Only)"**. This requires a verified Ubuntu host attestation and manual execution of exported bundles.
